> "가까운 노드부터 먼저, 먼 노드는 나중에 방문한다."

---

## 목표

> BFS(너비 우선 탐색)에 대해서 알아봅니다.

---

## 기본 원리

- 시작 노드로부터 **가까운 노드를 먼저** 방문하고, 멀리 떨어진 노드를 나중에 방문하는 순서로 진행한다.
- 위 과정에서 각 노드는 한번씩만 방문한다.

---

## Queue

#### 선형 큐

```python
from collections import deque

queue = deque([1, 2, 3, 4, 5])
queue.popleft()  # 맨 앞에서 꺼내기 → O(1), 빠름!
queue.popleft()

[_, _, 3, 4, 5] # 결과

# 다만, 파이썬의 deque는 `연결 리스트` 구조라 공간 낭비가 되지 않는다.
# 위 예는, 이해를 위한 예시이다.
```

- 일자로 된 배열에 `front`, `rear` 포인터를 두고 사용한다.
- 데이터를 꺼낼때마다 `front` 가 뒤로 밀려서, 앞쪽 공간이 버려진 채 낭비된다.

#### 원형 큐

- 배열의 끝과 시작을 논리적으로 연결해서 돌아가며 사용한다.
- `front` 가 끝에 도달하면 다시 0번 인덱스로 돌아온다.
- 공간 낭비가 없다.

---

## 코드

```python
from collections import deque

def bfs(graph, start):
    visited = []           # 방문 순서를 기록용
    queue = deque([start]) # 시작 노드를 큐에 넣음
    seen = {start}         # 이미 큐에 넣은 노드 표시

    while queue:
        node = queue.popleft()  # 큐의 맨 앞에서 꺼내기
        visited.append(node)

        # 이웃 노드들을 큐에 추가
        for neighbor in graph[node]:
            if neighbor not in seen:
                seen.add(neighbor)
                queue.append(neighbor)

    return visited

# 예시 그래프
#       1
#      / \
#     2   3
#    / \   \
#   4   5   6
graph = {
    1: [2, 3],
    2: [4, 5],
    3: [6],
    4: [],
    5: [],
    6: []
}

print(bfs(graph, 1))  # [1, 2, 3, 4, 5, 6]
```

- 덱을 쓰는 이유는 `front` 가 나올 때 O(1)이라 성능이 좋아서 사용한다.
- 대충 흐름을 보면,
    - queue 반복문을 돌려서 순서대로 queue에서 노드 하나를 빼고 visited에 추가한다.
    - 이후 해당 노드가 있는 graph에서 이웃한 노드들을 찾아 seen과 queue에 추가한다.
    - 여기서 중요한 점은, seen은 **set구조**라 중복 허용이 되지 않는다는 점이다.
    - 반복문이 끝나면, 방문한 순서대로 visited에 정렬되어 있다.
- ‘**한 명 나오면, 그 사람이 데려온 친구들이 줄에 합류한다. 줄이 빌 때까지**.’ 이 핵심을 이해하는 것이 중요하다.

## 복잡도

> V = 정점(Vertex) 개수, E = 간선(Edge) 개수, h = 그래프의 깊이, w = 그래프의 최대 너비
> 
- 시간 복잡도: **O(V+E)**
    - 모든 정점을 한 번씩 방문하고, 각 정점에서 인접한 모든 간선을 한번씩 확인한다.
- 공간 복잡도: **O(w)**
    - 현재 처리 중인 레벨 + 다음 레벨의 노드가 들어 있다.
    - 다만, Visited도 필요하기에 O(w+V)라고도 한다.
