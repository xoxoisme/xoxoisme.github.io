> "겹치는 결과를 별도의 메모리 영역에 저장해서 한 번만 계산하고 재사용한다."

---

## 목표

> DP(동적 계획법)에 대해서 알아봅니다.

---

## 기본 원리

- 하나의 큰 문제를 여러 개의 작은 문제로 나누어서 그 결과를 저장하여 다시 큰 문제를 해결할 때 사용하는 원리이다.
- 특정 알고리즘 보다는 하나의 문제해결 패러다임으로 볼 수 있다.
- DP를 **기억하며 풀기**라고 생각하면 편하다.

---

## 사용 이유

- DP는 일반적인 재귀 방식과 유사하다.
- 다만, 차이점은 일반적인 재귀를 단순히 사용 시 동일한 작은 문제들이 여러 번 반복되어 비효율적인 계산이 될 수 있다는 것이다.

#### 피보나치 수열을 활용하면 이해하는 데 편리하다.

- 피보나치 수열은 `1, 1, 2, 3, 5, 8, 13, 21, ...` 이다.
- 위를 재귀로 함수를 구성하면 `f(n) = f(n-1) + f(n-2)` 이 된다.
- 다만, 위는 `f(n-1)` 과 `f(n-2)` 를 각각 호출하면 동일한 값을 2번씩 구하게 되고 100번째를 구하면 기하급수적으로 증가한다.(매번 약 1.68 곱해짐)

#### 그래서 한 번 구한 결과 값을 저장해두고 재사용하면 100번째도 200회 내에 가능하다.

---

## 사용 조건

1. Overlapping Subproblems(겹치는 부분 문제)
    1. 동일한 작은 문제들이 **반복**하여 나타나는 경우에 사용 가능하다.
2. Optimal Substructure(최적 부분 구조)
    1. 부분 문제의 최적 결과 값을 사용해 전체 문제의 최적 결과를 낼 수 있는 경우를 의미한다.
    2. 부분 경로끼리 내부에서 **같은 노드를 공유하면 적용되지 않는다**.
    3. 즉, 이전의 값을 그대로 사용해도 결과가 똑같을 경우에만 가능하다.

---

## 구현 방법

#### Bottom-Up 방식

- 아래에서부터 계산을 수행하고 누적시켜서 전체 큰 문제를 해결하는 방식이다.
- `dp[0] -> dp[n]` 순으로 **반복문**을 통해 값을 전이시켜 재활용하는 방식이다.
- 순서가 명확하고 **전부 다 필요할 때** 사용한다.

```python
public static int bottomUp(int n){

    // 1. 바닥(0, 1)을 미리 저장 (재귀 없이 사전에 세팅)
    bottomup_table[0] = 0; bottomup_table[1] = 1;

    // 2. 아래서부터 위로 순서대로 채워나감 (재귀 없이 반복문!)
    for(int i=2; i<=n; i++){
        bottomup_table[i] = bottomup_table[i-1] + bottomup_table[i-2];
    }

    // 3. 최종 답 반환
    return bottomup_table[n];
}
```

#### Top-Down 방식

- 위에서부터 계산을 수행해 결과 값을 **재귀**를 통해 전이시켜 재활용하는 방식이다.
- `dp[n] -> dp[0]` 순이다.
- **필요한 값만 계산하면 될 때** 사용한다.
