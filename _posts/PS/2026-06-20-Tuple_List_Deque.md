## Tuple

- 모습은 리스트와 비슷하다.
- 다만, 튜플의 요솟값은 **변경이 불가**하다.
- 하나 중요한 건, **요소가 하나일 때는 반드시 콤마**를 붙여야한다.

```python
a = (1)         # bad 튜플 아님! 그냥 정수 1 (괄호는 계산용)
type(a)         # <class 'int'>

a = (1,)        # good 원소 1개짜리 튜플 (콤마 필수)
type(a)         # <class 'tuple'>
```

#### 메서드

```python
a = (1, 2, 2, 3, 2) # 메서드 쓰면 항상 초기화한다고 가정
# 튜플은 불변이라 append, remove, sort 등이 없음!

# count: 해당 값 개수 세기
a.count(2)      # 3

# index: 해당 값의 위치 (제일 먼저 찾은 것)
a.index(2)      # 1

# 인덱싱/슬라이싱은 됨 (읽기는 자유)
a[0]            # 1
a[1:3]          # (2, 2)

# 수정은 불가
a[0] = 100      # bad TypeError! 튜플은 못 바꿈
```

## List

- 데이터를 잘 관리할 수 있도록 **순서를 정해서 관리**하는 데이터 타입 중 하나이다.
- 시퀀스 데이터라고도 한다.(데이터에 순서가 존재)
- 그렇기에 리스트 안에 있는 데이터를 순서대로 접근이 가능하고, 인덱스를 통해 한번에 접근도 가능하다.

#### 인덱싱

```python
a = [1, 2, 3, 4, 5]

# 거꾸로 갈려면
a[-1] # 5
a[-5] # 1
```

#### 슬라이싱

```python
a = [1, 2, 3, 4, 5]

# 1~3까지만 원하면
a[0:3] # 인덱스 0, 1, 2만 필요함

# 4~5까지 원하면
a[3:5] # 인덱스 3, 4만 필요함

# 전체 원하면
a[:]

# n부터 끝까지
a[n:]

# n까지
a[:n]
```

- 마이너스도 가능하다.
- 깊은 복사이다.

#### 삭제

```python
a = [1, 2, 3, 4, 5]

del(a[2]) # [1, 2, 4, 5]

a = [1, 2, 3, 4, 5]

del(a[1:3]) # [1, 4, 5]
```

#### 메서드

```python
a = [1, 2, 3, 4, 5] # 메서드 쓰면 항상 초기화한다고 가정

# append: 값 이어 붙이기
a.append(6) # [1, 2, 3, 4, 5, 6]

# insert: 특정 위치에 추가
a.insert(0, 10) # [10, 1, 2, 3, 4, 5]
a.insert(3, 100) # [1, 2, 3, 100, 4, 5]

# remove: 특정 값 제거
a.remove(1) # [2, 3, 4, 5]

b = [1, 1, 2]
b.remove(1) # [1, 2] 제일 먼저 있는 값만 지워줌

# pop: 맨 마지막 값 반환 후 삭제
b = a.pop() # [1, 2, 3, 4]
b # 5

# extend: 리스트에 다른 리스트 연결
b = [6, 7]
a.extend(b) # [1, 2, 3, 4, 5, 6, 7]

# copy: 리스트를 복사
b = a.copy()
b # [1, 2, 3, 4, 5]

# reverse: 리스트 뒤집기
a.reverse() # [5, 4, 3, 2, 1]

# sort: 리스트 정렬
b = [3, 2, 1, 4, 5]
b.sort() # [1, 2, 3, 4, 5] 기본 오름차순 정렬
c = sorted(b) # [1, 2, 3, 4, 5] 원본이 아니라 새로 만들어 줌

# count: 리스트 내 해당 값 세기
b = [1, 1, 1, 2, 3]
b.count(1) # 3

# index: 리스트 내 해당 값의 위치
a.index(2) # 1

b = [1, 2, 4, 2, 5]
b.index(2) # 1 제일 먼저 찾은 값의 위치만 반환

# clear: 리스트 모든 값 삭제
a.clear() # []
```

- 그냥 `+` 와 extend의 차이점은 `+` 는 새로운 리스트를 만들어서 반환하는 것이고 extend는 원래 쓰던 객체에 붙이는 것이라 메모리 차이가 있다.
- 정렬할 때는 안에 다른 요소가 있으면 에러가 난다.

#### 얕은 복사/깊은 복사

```python
# 1 차원 리스트: 원시값 (immutable)만 포함
a = [1, 2, 3]
b = a[:]

# 메모리:
# a → [0x1000] → 1, 2, 3 (원시값 = 이미 독립) good
# b → [0x1100] → 1, 2, 3 (원시값 = 이미 독립) good

# 2 차원 리스트: mutable 객체 포함
a = [1, [2, 3], 4]
b = a[:]

# 메모리:
# a → [0x1000] → 1, [0x2000], 4
#                    │
# b → [0x1100] → 1, [0x2000], 4  bad
```

- 파이썬의 변수들은 **참조값만 저장**한다.
- `deepcopy()` 로 내부적으로 재귀를 돌며 모든 계층이 새 메모리를 생성하게 한다.
- 쉽게 겉에만 복사된다고 생각하면 된다.(내부는 공유)
- 중첩 리스트를 쓸 경우에 주의한다.
- `슬라이싱, copy()` 모두 얕은 복사에 포함된다.
