## 가상 메모리

> 실제 RAM보다 더 큰 메모리를 사용하는 것처럼 보이게 만드는 메모리 관리 기법이다.

<img src="/images/2026-03-06-메모리의 불연속 할당/스크린샷 2026-03-06 오후 2.19.09.png" alt="가상 메모리">

- 가상 주소를 통해 실제 메모리 주소를 **매핑**한다.
  - 이를 **MMU(Memory Management Unit)**가 페이지 테이블을 참조해 주소 변환을 한다.

### 가상 메모리를 사용하여 각 프로그램마다 독립적인 주소를 이용할 수 있다.

- 가상 메모리의 주소가 프로그램A, 프로그램B가 0x001로 같다하더라도 페이지 테이블로 인해 실제 메모리 주소는 다르다.
- 그래서 프로그램끼리 메모리 충돌이 없기에 보호할 수 있다.
- 서로의 메모리에 간섭을 할 수 없으니 당연하다.
- 개발자가 편하게 프로그램을 개발할 수 있다.
- 메모리 관리에 효율이 좋다.

---

## 페이지 폴트

> 프로세스의 주소 공간에는 존재하나, 실제 RAM에는 존재하지 않을 때 발생한다.

#### 1. 프로세스의 주소 공간인 Page에 접근하면 MMU가 트랩을 발생시킨다.

#### 2. 운영체제는 CPU의 동작을 잠시 멈춘다.

#### 3. 운영체제는 트랩이 발생한 Page를 가상 메모리를 찾는다.

#### 4. 해당 Page를 실제 메모리의 비어있는 Frame에 디스크에 접근해서 로드한다.

#### 5. 페이지 테이블을 최신화한다.

#### 6. 중단되었던 CPU를 다시 실행시킨다.

- 위 과정은 **CPU를 중단시킨 후**, 진행된다. 이 포인트가 중요하다.
- 4번의 디스크에 접근하는 건 SSD(μs), HD(ms) **각각 수백 마이크로 초, 밀리초**가 걸리게 된다.
- CPU는 보통 1클럭 당 0.3(ns) **나노초**가 걸린다.
- 페이지 폴트가 발생하면 CPU는 디스크에 접근하는 시간 때문에 너무 지체된다.
- 그래서, Workding Set과 PFF(Page Fault Frequency)를 통해 이를 해소한다.

---

## 메모리 할당

> 연속 할당과 불연속 할당이 중요한데, 현대 운영체제는 대부분 불연속 할당을 사용한다.

### 페이징

> 현재 운영체제는 테이블 1개만 참조하는 순수 페이징 기반을 많이 사용한다.

- 이전에 설명한 방법으로 동일한 크기(4kb)로 페이지 단위를 나눠 메모리의 서로 다른 위치에 프로세스를 할당한다.
- 홀의 크기가 균일하지 않은 문제가 해소되지만, 주소 변환이 복잡해진다.

### 세그멘테이션

![스크린샷 2026-03-06 오후 3.01.41](/images/2026-03-06-메모리의%20불연속%20할당/스크린샷%202026-03-06%20오후%203.01.41.png)

- 페이지 단위가 아닌 논리 단위인 **세그먼트로 나누는 방식**이다.
- **코드/데이터/스택/힙** 영역으로 나누어 프로세스를 할당한다.
- 논리 단위로 세그먼트를 나누면 각 세그먼트가 가져야 하는 용량이 정해진다. 그러면 여러 프로세스가 종료 후 재 배치를 진행하고 나면 RAM의 메모리가 중간 빈 공간에 세그먼트를 넣어야만 하는데 이 때 **외부 단편화**가 발생한다.

#### 메모리를 나눈 크기보다 프로그램이 커서 들어가지 못하는 공간이 발생하는 현상을 외부 단편화라고 한다.

- **홀(빈 공간) 크기**가 균일하지 않다는 단점이 존재한다.
- 하지만, 논리 단위 별로 권한을 줄 수 있다는 것 만으로 **공유와 보안 측면에서 장점**이 있다.

### 페이지드 세그멘테이션

- 위 두가지를 합친 방법으로 의미 단위인 세그먼트로 나누고, 임의의 길이가 아닌 동일한 크기(4kb)의 페이지를 프레임에 배치한다.
- 하지만, **메모리 접근을 위해 최소 2번** 이상이기에 성능이 떨어진다.
