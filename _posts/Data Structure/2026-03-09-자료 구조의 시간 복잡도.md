## 목표

> 배열, 스택, 큐, 이중 연결 리스트 각각의 특징과 시간 복잡도에 대해서 알아봅니다.

---

## 배열

- **같은 타입의 변수들이 연속된 메모리**로 저장되어 있는 집합이다.
- 중복을 허용하고, 순서가 있다.

### 시간 복잡도

| 접근 | 탐색 |          삽입          |          삭제          |
| :--: | :--: | :--------------------: | :--------------------: |
| O(1) | O(n) | 끝 : O(1), 중간 : O(n) | 끝 : O(1), 중간 : O(n) |

- 접근은 `address` = `base_address` + `(index × element_size)` 이기에 **인덱스만 알면** 해당 위치로 바로 접근이 가능하다.
- 탐색은 순차적으로 탐색한다. 다만, 정렬되어 있다면 **이진 탐색법**이 가능하기에 `O(logn)`이다. 
- 삽입과 삭제는 **사이즈를 알기에** 마지막 위치에 바로 접근이 가능하나, 중간은 해당 위치 이후의 데이터를 **순차적으로 밀거나 댕겨야** 한다.
- 임의의 인덱스에 해당하는 데이터에 접근하는 **랜덤 접근에 유용하다**.

## 스택

> 보통 접근은 임의 접근을 기준으로 하나, 허용된 연산만을 기준으로 하는 책은 다를 수 있다.

- **후입선출(LIFO, Last In First Out)** 구조의 자료구조이다.
- 가장 **최근에 삽입된 데이터가 가장 먼저 제거**된다.
- 데이터의 삽입과 삭제는 **한쪽 끝(top)** 에서만 이루어진다.

### 시간 복잡도

|             접근(peek)             | 탐색 | 삽입(push) | 삭제(pop) |
| :--------------------------------: | :--: | :--------: | :-------: |
| `top`접근 : O(1), 임의 접근 : O(n) | O(n) |    O(1)    |   O(1)    |

- 스택은 `top` **위치만 관리**하기 때문에 삽입(push), 삭제(pop)는 항상 `O(1)` 이다.
- 임의의 위치에 접근하는 **랜덤 접근은 지원하지 않는다.** 스택의 목적이 최근 데이터를 처리하기 위함이기 때문이다.
- 탐색을 하려면 **위에서부터 순차적**으로 확인해야 하기 때문에 `O(n)` 이다.
- 배열이나 연결 리스트를 이용해 구현할 수 있으며, 대부분 `top` **포인터(또는 index)**로 현재 위치를 관리한다.

## 큐

- **선입선출(FIFO, First In First Out)** 구조의 자료구조이다.
- **먼저 들어온 데이터가 먼저 제거**된다.
- 데이터는 **rear(뒤)** 에서 삽입되고 **front(앞)** 에서 삭제된다.


### 시간 복잡도

|                 접근                 | 탐색 | 삽입(enqueue) | 삭제(dequeue) |
| :----------------------------------: | :--: | :-----------: | :-----------: |
| `front`접근 : O(1), 임의 접근 : O(n) | O(n) |     O(1)      |     O(1)      |

- 큐는 **front와 rear 위치를 관리**하기 때문에 삽입과 삭제는 항상 `O(1)` 이다.
- 삽입은 **rear 위치에 바로 추가**된다.
- 삭제는 **front 위치의 데이터를 제거**한다.
- 임의의 위치에 바로 접근하는 **랜덤 접근은 지원하지 않는다.**
- 특정 데이터를 찾으려면 **front부터 순차적으로 탐색해야 하므로** `O(n)` 이다.
- 배열 또는 연결 리스트로 구현할 수 있으며, **front / rear 포인터(또는 index)** 를 이용해 위치를 관리한다.

### 주요 연산

|     연산     |            설명             |
| :----------: | :-------------------------: |
|   enqueue    | 큐의 뒤(rear)에 데이터 삽입 |
|   dequeue    | 큐의 앞(front) 데이터 제거  |
| peek / front |   큐의 맨 앞 데이터 확인    |
|   isEmpty    |    큐가 비어있는지 확인     |

## 이중 연결 리스트

- 각 노드가 **두 개의 포인터(prev, next)** 를 가지는 연결 리스트이다.
- **이전 노드(prev)** 와 **다음 노드(next)** 를 모두 가리킨다.
- 단일 연결 리스트와 달리 **양방향 탐색이 가능**하다.


### 시간 복잡도

| 접근 | 탐색 | 삽입 | 삭제 |
| :--: | :--: | :--: | :--: |
| O(n) | O(n) | O(1) | O(1) |

- 이중 연결 리스트는 **연속된 메모리가 아니기 때문에 랜덤 접근이 불가능**하다.
- 특정 위치에 접근하려면 **head 또는 tail에서부터 순차적으로 이동**해야 하므로 `O(n)` 이다.
- 노드의 위치를 알고 있다면 **포인터(prev, next)만 변경하면 되므로** 삽입과 삭제는 `O(1)` 이다. 위치를 모른다면 순차적으로 이동해야하기에 `O(n)`이 된다.
- 단일 연결 리스트와 달리 **이전 노드를 바로 알 수 있기 때문에 삭제가 더 편리하다.**

### 각 노드 구조

- `prev` : 이전 노드를 가리키는 포인터
- `data` : 실제 데이터
- `next` : 다음 노드를 가리키는 포인터
